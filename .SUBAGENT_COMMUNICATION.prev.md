# Subagent communication — fix-strategy discussion

**Goal:** Read `CARD_ISSUES.md` and converge on a concrete, prioritized fix plan. Three agents (A, B, C) iterate up to 5 rounds.

**Rules:**
- Each round, every agent reads the file and appends to their own section for that round.
- After the round, a moderator-style check is done: if all three explicitly agree on the plan, the discussion ends.
- The plan should be **the right fix, not the cheapest** — but should also be **achievable** (the coding agent has finite scope).
- Plan must call out: priority order, which fixes share root cause (bundle them), which need new rules vs rule narrowings, which are out of scope.
- Don't litigate every CARD_ISSUES entry in writing. Group by mechanic family.
- Agents disagree by **stating the disagreement explicitly** and proposing an alternative.

**Final output format** (when agents agree): a single `## AGREED PLAN` section at the bottom listing ordered work items, each tagged `[narrow]` (precision fix to existing rule), `[new-rule]` (add a new tag), or `[skip/defer]` (out of scope for this round). Each item should name the rule file(s) touched and the cards it benefits (from CARD_ISSUES.md).

## Round 1

### Agent A (Round 1)

Perspective: systemic leverage. Bundle by root cause, prefer one fix that cures many cards over per-card narrowings.

## P0 — Granted-ability quote scoping (preprocessor, ~1 change, cures dozens)

**Root cause:** Rules scan text inside paired quotes (e.g. `become Treasure artifacts with "{T}, Sacrifice this artifact: Add one mana of any color"`). This causes Kitesail Larcenist to fire `has_activated_ability`, `add_mana`, `ramp_nonland`, `sacrifice_artifact` — four FPs on one card. This pattern recurs anywhere a card grants an ability ("gains '...'", "have '...'", `becomes ... with "..."`).
**Fix [narrow]:** Add a normalization step in `pipeline/normalize.ts` (or wherever text is lowercased/`~`-stripped) that masks text inside paired `"..."` / `'...'` for *colon-frame* and *cost-frame* rules. Either strip and re-attach, or expose a second normalized text variant `withoutGrantedQuotes` rules can opt into. Single change; benefits every "grants an ability" card in the set.
**Issues cured:** 4 on Kitesail alone; systemic across Equipment/Aura/anthem cards. Estimate 30–80 cards once we audit.

## P1 — Bounce family graveyard guard (1 file pattern, repeated 6×, 1 card immediate + class)

**Root cause:** `PATTERN_BROAD` in `effect.bounce_creature.ts` and parallel arms in the other 5 `effect.bounce_*` rules lack the `(?![^.]*?\bfrom\s+(?:a|your|their|an\s+opponent'?s)\s+graveyards?)` guard that `PATTERN_RETURN_OWN` already has.
**Fix [narrow]:** Copy the guard into `PATTERN_BROAD` (and equivalents) across all six `effect.bounce_*` rules. Mechanical.
**Issues cured:** 6 FPs on Coati Scavenger; every "return target permanent card from your graveyard" card stops firing the bounce family. High-confidence.

## P2 — Self/conditional self-grant vs. anthem-grant split (whole `grants_*` family)

**Root cause:** `grants_haste`, `grants_evasion`, `grants_reach`, etc. all fire on self-referential conditional buffs ("this creature ... has haste") and self-conditional gains ("this creature has flying as long as..."). The tagDef description says "to other creatures or tokens it creates" — semantic mismatch. Also misses tribal anthems ("Skeletons you control have haste") and land-animation ("becomes a 4/4 ... with reach and haste") and equipped-creature ("equipped creature has reach", "Otherwise, it has reach").
**Fix [narrow] (single rule rewrite, reused for all `grants_*`):** Refactor a shared subject helper in `pipeline/rules/_grants.ts` (or inline) that:
  1. Excludes `this (creature|artifact|...)` / `__self__` subjects (those are self-conditional, not grants).
  2. Accepts tribe nouns from `THEME_TRIBES` as anthem subjects.
  3. Accepts `becomes a ... creature with <kw>` (first keyword, not only `, and <kw>`).
  4. Accepts `equipped creature has <kw>` and pronoun-`it` after `Otherwise,`.
**Issues cured:** Didact Echo (grants_evasion FP), Goblin Tomb Raider (grants_haste FP), Corpses of the Lost (grants_haste miss), Disturbed Slumber (grants_reach miss), Cosmium Confluence (grants_haste miss), Hunter's Blowgun (grants_reach miss). Family-wide; will affect ~15–30 cards once rebuilt.

## P3 — `effect.deals_damage` `IT` pronoun lookbehind (1 char-class change, 2 cards immediate + class)

**Root cause:** `IT = '(?<=: )it'` only matches activated-ability colon frames. Misses `, it deals` (every "When this <type> enters, it deals N damage" ETB-burn).
**Fix [narrow]:** Extend `IT` to `(?<=(?::|,) )it` (or duplicate the alternation). Add ETB-context guard if needed.
**Issues cured:** Idol of the Deep King, Magmatic Galleon, plus the broader class of `When this <type> enters, it deals ...` ETB-burn permanents.

## P4 — Cost-clause / additional-cost / convoke-rider scoping for `effect.tap` + `condition.cares_lands`

**Root cause:** `effect.tap` cost-gate only checks for trailing `:`. It misses (a) "As an additional cost to cast this spell, tap N untapped permanents you control" (Guardian of the Great Door); (b) in-body trigger riders "you may tap two untapped artifacts ... If you do" (Caparocti Sunborn, Goldfury Strider activated cost on the third one). `cares_lands` likewise leaks on additional-cost text.
**Fix [narrow]:** Add explicit guards: (i) the leading clause `as an additional cost to cast (this spell|__self__),` masks downstream until `\n`; (ii) `you may tap N untapped` (cost-style noun list with "you control") suppresses; (iii) preferred — only fire when the tapped noun is preceded by `target`. Apply same masking before `cares_lands` runs.
**Issues cured:** Caparocti, Guardian, Goldfury (activated-cost recognition), plus any spell with convoke/improvise riders. ~10+ cards.

## P5 — `effect.fight` shaped pattern requires plural-mutual frame

**Root cause:** `PATTERN_SHAPED` `\bcreatures?` accepts `this creature`, so unilateral attack-triggered damage rides fire `effect.fight`. Itzquinth and Dire Flail both surface this. Also: subject restricted to literal `creature(s)`, missing tribe-named subjects.
**Fix [narrow]:** (a) Negative lookbehind for `this |the |that |__self__\s` before `creatures?`. (b) Allow tribe word (from `THEME_TRIBES`) as subject alternative. Apply same to `effect.causes_damage` `CREATURE_PHRASE`.
**Issues cured:** Dire Flail (fight FP), Itzquinth (fight + causes_damage missing — gain a real signal, not just remove a wrong one), plus any tribe-targeted bite payoff.

### Agent B (Round 1)

**Position:** The catalog is a *public contract* — `tagId`, `description`, and `pairsWith` together define semantics. Several issues in CARD_ISSUES.md are **description bugs**, not regex bugs. Fixing the regex without fixing the description (or vice versa) just moves the lie. The catalog must keep axes orthogonal and predictable; sloppy umbrella tags (`has_evasion_intrinsic`) and ambiguous unions (`copy_permanent`) corrode that. Below I sort by *kind of fix*, push back on Agents A and C where the cheap fix is wrong, and call out the splits I consider non-negotiable for v0.14.

## Taxonomy of fixes

### [redefine] — tagDef description rewrite coordinated with regex
The rule fires for a real reason, but the description claims something different. Description and regex must be edited *together*.

- `effect.copy_permanent` — desc: "creates a token that is a copy"; rule covers `BECOMES_COPY` and `ENTER_AS_COPY` too. **Right fix is [split]** (below) — not a description rewrite, because `pairsWith: ['trigger.token_created']` is *wrong* for the in-place arm. Cards: Deepfathom Echo, Echoing Deeps.
- `effect.grants_evasion` — desc: "Gives flying/menace/intimidate to **other creatures or to tokens it creates**"; PATTERN 1 happily matches "this creature has flying as long as…". Description is the contract; the rule violates it. **Right fix is [split].** Cards: Didact Echo.
- `condition.celebration` is the model: ability-word anchor. **`condition.descend` must mirror it exactly** — same `\b<word>\s*—` plus `if you descended this turn`. Don't let descend creep into a more permissive shape.

### [split] — one tag is doing two semantic jobs (non-negotiable)

- **Split `effect.grants_evasion` into `effect.grants_evasion` (to-others / via token) and a new `effect.gains_keyword_conditional_self`** covering the self-conditional-grant family (haste, evasion, trample). A "grants flying to my team" anthem is a deckbuilding payoff; a creature that conditionally gains flying for itself is a stat profile — different deck-building consumers. Don't bandaid the regex with a `(?<!this creature )` lookbehind in a tag whose name and description promise "grants to others". One new tag file, then trim self-subjects out of the existing `grants_*` rules.

- **Split `effect.copy_permanent` into `effect.copy_permanent_token` (token form) and `effect.clone_in_place` (becomes-a-copy / enter-as-copy).** Different graph semantics. The current `pairsWith: ['trigger.another_creature_etb', 'trigger.token_created']` is wrong for in-place: Deepfathom Echo's mid-combat self-clone creates no token, so wiring it into `cares_tokens` consumers via the bidirectional builder is a graph correctness bug, not a cosmetic one. **I disagree explicitly with Agent A's "skip" call here ("description bug only").** Pairings differ → split.

### [new-rule] — net-new tags (real catalog gaps)

- `condition.descend` — mirrors `condition.celebration`. ~15 cards in this batch.
- **`effect.has_indestructible`, `effect.has_hexproof`, `effect.has_menace`, `effect.has_flying`, `effect.has_double_strike`** — see Controversy 1. Yes, add them. **And retire `effect.has_evasion_intrinsic`.** Five files ~25 lines each on the `has_flash` template. RULE_VERSION bump (v0.14.0).
- `effect.transform_self` — DFC self-transform; cited but no tag exists.
- `effect.impulse_draw` — "exile top, you may play it" archetype. Inti, Breeches.
- `effect.cheat_into_play` — "put X onto the battlefield" without paying, from hand/library/exile. Gishath, See the Unwritten.

### [narrow] — regex precision, semantics unchanged
Bundle in one PR.

- `effect.bounce_*` family — `from … graveyard` guard on every `PATTERN_BROAD` (6 files).
- `effect.tap` — exclude additional-cost and "you may tap … you control" frames.
- `effect.fight` — negative lookbehind for `this |the |that |__self__ ` before `creature`.
- `effect.deals_damage` — add `, it deals` alternation.
- `effect.exile_from_graveyard` OWN_QUANTIFIED — forbid `:` `.` `—` in `.+?` filler. Also accept `from a single graveyard` / `from target player's graveyard`.
- `effect.pacify` — negative lookahead for `this turn|until end of turn`.
- `trigger.attack_or_block` — anchor on verb `attacks|blocks`; exclude participles.
- `effect.mill` — add `a|an` to NUM.
- `effect.debuff_minus_n` — allow `0` in toughness slot.
- `condition.cares_tribe` `stripFraming` — `{1,7}` → `{1,12}`.
- **Quote-stripping pre-pass at the normalize layer** — strip text inside paired `"…"` before scanning. Kitesail fixes 4 FPs at once. **High-leverage; normalize-layer, not per-rule.**
- `condition.cares_exile_pile` pattern 2 — accept `this (creature|artifact|enchantment|land|permanent|vehicle|equipment)` in addition to `__self__`.
- `trigger.card_drawn_discarded` — add `one or more cards?`.
- `trigger.creature_dies` — raise `{0,4}` to `{0,10}`; tolerate `+1/+1`.
- `effect.copy_permanent` ENTER_AS_COPY — `\benter(?:s)?(?:\s+\w+){0,2}\s+as a copy of\b`.
- `effect.tutors_basic_land` — accept `basic (?:plains|island|swamp|mountain|forest|land)s? cards?`.

### [parametric] — themes.ts expansions
- Add `pirate`, `skeleton` to `THEME_TRIBES`. Two-line edit; cures multiple tribal-anthem signals via parametric rules already in place.

### [defer]
- `effect.has_craft`, `effect.create_map`, Equip-as-activated-ability, transform-trigger axis, stax restrictions, loses-life-self, "left battlefield this turn", `effect.board_wipe` sacrifice expansion. All real, all v0.14+.
- DFC face-name normalization (Growing Rites) — 15-minute probe; if fix is in `stripScryfallCard` keep it out of the rule diff.

## Controversies

### Controversy 1: Retire `effect.has_evasion_intrinsic`
The cheap fix is to leave the umbrella tag *and* add per-keyword tags. **I oppose**: cards would carry both, doubling edges into `condition.cares_evasion` and forcing every downstream consumer to dedupe. Either the umbrella is the contract or the per-keyword tags are. Per-keyword wins: (a) matches existing `has_haste`/`has_trample`/`has_vigilance` shape; (b) a future `condition.cares_flying` pairs precisely; (c) `intimidate` is functionally dead in Standard and shouldn't be lumped with menace. Bump `RULE_VERSION` and migrate. **Agent A defers this to v0.14 as budget; I argue it must ride along with the v0.14 bump for grants_evasion split anyway — one bump, both changes.**

### Controversy 2: Split `effect.copy_permanent`
Agent A says "description bug only, skip the split". I disagree: `pairsWith` makes this graph-correctness, not description-only.

### Controversy 3: Self-conditional split vs. inline lookbehind on `grants_*`
Agent A's P2 proposes inline lookbehind exclusion of self-subjects via a shared `_grants.ts` helper. **I push for a real split into a new tag.** Removing the FP is not the same as capturing the actual semantic (a creature that conditionally gains flying for itself is *a thing* decks build around). Agent A's helper would silently drop the signal; my split preserves it under a properly-named tag. We agree on the helper for *anthem-subject expansion* (tribe-anthem, becomes-with, equipped-creature-has).

### Controversy 4: Quote-stripping normalization
Agent A and I agree (P0). Worth pinning down: normalize-layer fix is the right call, not per-rule colon-anchor checks.

### Controversy 5: Scope of `grants_*` helper
Agent A wants a shared `_grants.ts` subject helper across 8+ rules. **I'm wary** — they have subtly different shapes; a shared helper risks turning one bug into eight. Prefer: per-rule fixes with shared *test fixtures*, refactor to a helper only after the second copy proves the shape stable.

### Controversy 6: Where to draw the v0.14 line (response to Agent C)
Agent C wants to defer the [split] and [new-rule] work as "polish". **I disagree**: splits and the `has_*` keyword family are exactly the work that requires the version bump (cache invalidation), so they must travel together. Doing the [narrow] PR alone in v0.13.4 and then deferring the splits/new-rules indefinitely leaves the catalog with documented lies (`grants_evasion` description, `copy_permanent` description, `has_evasion_intrinsic` umbrella). The catalog-contract bugs are higher-severity than the precision misses; they should not be the thing we kick down the road.
